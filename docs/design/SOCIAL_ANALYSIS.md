# 百世江湖 社会态势分析与项目重新定位

## 一、2026年中国社会情绪地图

### 1.1 核心社会情绪

| 情绪 | 表现 | 与项目的关联 |
|------|------|-------------|
| **内卷疲惫** | 996/007常态化，年轻人"躺平"心态蔓延 | 转生=重新来过，是最深层的逃避现实需求 |
| **身份焦虑** | 阶层固化感强，"努力不一定有用"的无力感 | 多世轮回=多次机会，打破"一次定终身"的焦虑 |
| **意义感缺失** | 物质满足后精神空虚，"活着为了什么" | 每一世的选择累积成传奇，给存在赋予意义 |
| **社交孤独** | 线上社交繁荣但深度连接稀缺 | NPC记住你=被世界认可，满足归属感需求 |
| **怀旧情结** | 对"简单时代"的怀念，武侠情怀 | 武侠题材+文字驱动，回归叙事本质 |

### 1.2 情绪需求矩阵

```
                    高代入感
                       ↑
         ┌─────────────┼─────────────┐
         │  赘婿/逆袭   │  百世江湖    │
         │  (代入体验)  │  (轮回认同)  │
    低───┼─────────────┼─────────────┼───高
    深度 │  系统流/爽文  │  模拟经营类  │  深度
         │  (情绪释放)  │  (系统探索)  │
         └─────────────┼─────────────┘
                       ↓
                    低代入感
```

**百世江湖的独特位置**: 高代入感 + 高深度 = 市场上几乎没有的组合

### 1.3 为什么"轮回"题材现在爆火

番茄小说数据佐证：
- **穿越**: 3.3亿覆盖（#1）
- **重生**: 1.2亿覆盖（#3）
- **末世**: 3585.7万覆盖（#11）
- **年代**: 持续走高

**本质**: 2026年的年轻人想要"第二次机会"。不是简单的复活，而是带着记忆和反思重新活一次。

## 二、竞品分析与差异化

### 2.1 现有"轮回"类游戏

| 游戏 | 机制 | 缺陷 | 百世江湖的机会 |
|------|------|------|-------------|
| 黑帝斯 | Roguelike+叙事 | 轮回是关卡机制，不是核心体验 | 我们的轮回是身份核心 |
| 暴走英雄坛 | 武侠+轮回 | 简单的重生机制，无记忆系统 | 我们有跨世代记忆 |
| RimWorld | 生存模拟 | 无转生概念 | 我们有身份传承 |
| 鬼谷八荒 | 修仙模拟 | 单一周目，无轮回 | 我们有多世累积 |

### 2.2 差异化定位

**一句话**: "RimWorld的世界模拟 × 武侠的身份认同 × Roguelike的轮回快感"

**核心卖点排序**:
1. 世界记住你（记忆系统）← 最核心
2. 每一世都不同（随机性+选择累积）
3. 成长感（跨世代积累）
4. NPC关系网（社交模拟）
5. 武侠世界观（题材包装）

## 三、情绪驱动的设计原则

### 3.1 四种核心情绪

| 情绪 | 触发场景 | 设计实现 |
|------|---------|---------|
| **被记住** | NPC说出前世的事 | MemorySystem + 跨世代记忆 |
| **成长感** | 属性/声望跨世累积 | ReincarnationSystem |
| **选择重量** | 一世的选择影响百世 | 事件系统 + 因果链 |
| **命运感** | 随机出生+前世影响 | 转生算法 + 属性继承 |

### 3.2 情绪曲线设计（单世）

```
情绪强度
  ↑
  │     ★前世记忆被唤醒
  │    / \
  │   /   \    ★建立势力
  │  /     \  / \
  │ /       \/   \    ★死亡
  │/              \  / \  /★转生
  ┼────────────────────────→ 时间
  出生   成长   巅峰  衰老  死亡  转生
```

### 3.3 反面教材（不要做的事）

- ❌ 不要做成"又一个武侠RPG"
- ❌ 不要做成"AI聊天游戏"
- ❌ 不要做成"纯文字MUD"
- ❌ 不要做成"数值模拟器"
- ✅ 要做"一个会记住你的世界"

## 四、Steam市场机会

### 4.1 Steam独立游戏现实

- 2025年新发行19,000+游戏
- 中位数收入仅$249
- 收入<$1,000的游戏占66%
- **成功关键**: 进入前5%

### 4.2 百世江湖的Steam优势

| 优势 | 说明 |
|------|------|
| **题材独特** | 市面上没有"轮回武侠模拟器" |
| **可传播性** | "我活了100世" 是天然的分享点 |
| **低成本** | 文字+简单图形，不需要3A美术 |
| **长尾潜力** | 系统深度驱动的长期留存 |
| **中国风** | 武侠题材在Steam有稳定受众 |

### 4.3 Steam宣传语候选

3. > "每一次死亡，都是新的开始。"

## 五、MVP重新定义

### ❌ 旧MVP（错误方向）
```
1. 角色创建
2. 基础移动 + 地图
3. 对话系统
4. 第一个NPC
5. 基础战斗
```

### ✅ 新MVP（正确方向）
```
第一优先级：Memory Engine v0
  - NPC记住玩家行为
  - 跨世代记忆继承
  - 世界历史记录

第二优先级：Reincarnation v0
  - 死亡 → 转生流程
  - 属性继承/丢失
  - 前世记忆碎片

第三优先级：Text Simulation v0
  - 文字版江湖模拟
  - 选择 → 后果
  - 时间推进

第四优先级：NPC AI v0
  - 基础人格模型
  - 关系值系统
  - 记忆驱动对话
```

### 验证闭环
```
玩家做选择
    ↓
NPC记住选择
    ↓
玩家死亡
    ↓
玩家转生
    ↓
NPC认出玩家
    ↓
"这个感觉对了"
```

**如果这个闭环跑通，项目就成功了80%。**

## 六、技术架构重新排序

### 旧架构（以战斗为核心）
```
World → Character → Memory → Faction → Combat → Economy → Reincarnation → AI
```

### 新架构（以记忆为核心）
```
Memory → Reincarnation → World → Character → AI → Faction → Economy → Combat
```

**核心变化**:
- Memory系统成为一切的基础
- Reincarnation是第二个核心
- Combat降到最低优先级
- AI服务于记忆和转生
