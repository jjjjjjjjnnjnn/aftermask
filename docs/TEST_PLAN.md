# Legacy Retention Rate 测试方案

> 版本: 0.1 | 创建: 2026-06-18 | 目标: 验证核心循环

## 核心指标

### Legacy Retention Rate (LRR)

```
LRR = 想开第三世的人数 / 测试总人数
```

**判断标准**:
- LRR ≥ 80%: 核心循环成立，继续开发
- LRR 50-79%: 有潜力，需要优化
- LRR < 50%: 核心循环失败，需要重设计

## 测试目标

**唯一目标**: 验证玩家是否因为「改变世界」而想开第三世。

**不是验证**:
- 美术质量
- 音效质量
- UI设计
- 战斗系统
- 代码质量

**只验证**:
- 第一世结束后，玩家是否想开第二世
- 第二世看到遗产时，玩家是否有感觉
- 第二世结束后，玩家是否想开第三世

## 测试对象

### 必须是陌生人

**原因**:
- 朋友会说「不错」「挺好」「加油」
- 陌生人会说「没感觉」「看不懂」「为什么要继续玩」

### 寻找渠道

1. **Reddit**: r/playmygame, r/indiegaming, r/gamedev
2. **Discord**: Indie Game Developers, Playtest Exchange
3. **Twitter/X**: #indiedev, #gamedev, #playtesting
4. **Steam**: Next Fest Demo（如果进入该阶段）
5. **朋友的朋友**: 通过朋友介绍，但不能是直接朋友

### 筛选条件

- 喜欢模拟经营类游戏（RimWorld, Dwarf Fortress, Crusader Kings）
- 喜欢叙事驱动游戏（Disco Elysium, Pentiment）
- 愿意花30分钟试玩
- 能用英文沟通

### 目标数量

- **最低**: 10人
- **理想**: 20人
- **上限**: 30人（超过后反馈重复）

## 测试流程

### 步骤1: 发送试玩邀请

**邀请模板**:

```
Subject: Playtest: A reincarnation simulation (30 min)

Hi [Name],

I'm developing a indie game called "Hundred Lives Jianghu" — 
a reincarnation simulation where every life leaves a mark on the world.

I'm looking for playtesters to try a 30-minute text prototype.

What you'll experience:
- Live one life, make choices, die
- Be reborn, see how your actions changed the world
- Decide if you want to live a third life

What I need from you:
- 30 minutes of your time
- Honest feedback (brutal honesty preferred)
- Answer 5 questions after playing

No coding, no downloads — just a text file you read through.

Interested? I'll send you the prototype link.

Best,
[Your Name]
```

### 步骤2: 发送原型文件

**发送内容**:
- `tests/legacy_prototype_v1.md` 文件
- 测试问卷链接（Google Forms或Typeform）
- 说明：纯文本，30分钟，读完回答5个问题

### 步骤3: 玩家试玩

**玩家操作**:
1. 打开原型文件
2. 按照指示阅读
3. 在CHOICE点做出选择
4. 读完整个原型
5. 回答问卷

**玩家不需要**:
- 下载任何东西
- 安装任何软件
- 运行任何代码
- 创建任何账号

### 步骤4: 收集反馈

**问卷链接**: [待创建]

**问卷内容**: 见下方

### 步骤5: 分析数据

**分析指标**:
- Legacy Retention Rate (LRR)
- 情感反应分布
- 无聊时间点
- 开放反馈主题

## 测试问卷

### 问题1: 第一世结束后

> 第一世结束后，你想开第二世吗？

- [ ] 是，我很好奇接下来会发生什么
- [ ] 是，我想尝试不同的选择
- [ ] 否，我已经失去兴趣
- [ ] 不确定

**这是关键指标之一。**

---

### 问题2: 第二世开始后

> 第二世开始后，你最想知道什么？

开放回答。

**分析**: 玩家是否对遗产有好奇心？

---

### 问题3: 看到前世遗产时

> 看到前世留下的遗产时，你的感受如何？

- [ ] 无感，这和我无关
- [ ] 有趣，有点意思
- [ ] 惊喜，我想了解更多
- [ ] 震撼，这改变了一切

**这是关键指标之一。**

---

### 问题4: 什么时候最无聊

> 在整个体验中，什么时候你最无聊？

开放回答。

**分析**: 哪些部分需要优化？

---

### 问题5: 第二世结束后

> 第二世结束后，你想开第三世吗？

- [ ] 是，我非常想
- [ ] 是，但没那么急切
- [ ] 否，我已经满足了
- [ ] 否，我已经失去兴趣
- [ ] 不确定

**这是关键指标之一。Legacy Retention Rate = 选「是」的比例。**

---

### 额外问题（可选）

> 你会如何用一句话向朋友描述这个游戏？

开放回答。

**分析**: 玩家是否能理解卖点？

---

> 这个游戏最吸引你的是什么？

开放回答。

**分析**: 核心卖点是否被感知？

---

> 你愿意花多少钱买这个游戏？

- [ ] $0 (免费也不玩)
- [ ] $5以下
- [ ] $5-15
- [ ] $15-25
- [ ] $25以上

**分析**: 付费意愿。

---

> 你玩过哪些类似的游戏？（RimWorld, Dwarf Fortress, Crusader Kings等）

开放回答。

**分析**: 目标用户匹配度。

## 决策矩阵

### LRR ≥ 80%

**结论**: 核心循环成立

**下一步**:
1. 继续开发Legacy System v1
2. 进入Godot原型阶段
3. 开始Steam商店页面准备

### LRR 50-79%

**结论**: 有潜力，需要优化

**下一步**:
1. 分析开放反馈
2. 找出无聊时间点
3. 优化Legacy Prototype v2
4. 重新测试

### LRR < 50%

**结论**: 核心循环失败

**下一步**:
1. 完全重新设计核心循环
2. 或放弃项目
3. 分析失败原因，用于下一个项目

## 成功标准

**必须达到**:
- LRR ≥ 80%
- 至少10人完成测试
- 至少3人给出「震撼」评价

**如果未达到**:
- 分析原因
- 优化原型
- 重新测试
- 如果连续两次未达到，重新设计核心循环
